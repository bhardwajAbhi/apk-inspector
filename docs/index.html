The Problem

APK analysis takes too many steps

Analyzing an APK usually means switching between the terminal, multiple command-line tools, a browser for hash lookups, and separate windows for certificates and permissions. APK Inspector brings that workflow into Nautilus via a single Scripts menu action.

🔧

Multiple tools

Metadata, signing, certificates, and contents often require separate commands and parsers.

⏱

Slow triage

Copying hashes, checking permissions, and reviewing manifests adds friction to every sample.

📂

Disconnected workflow

The APK is in your file manager, but the analysis tools live elsewhere on your system.

The Solution

One click. Structured results.

Right-click any .apk file in Nautilus, open Scripts → Get APK Details, and a clean GTK window opens with organized tabs for every major analysis area.

🗂

File Info

Size, MD5, SHA-256. Copy hashes in one click.

🔐

Permissions

All declared permissions. Dangerous ones highlighted.

✍

Signing & Certs

Signature schemes, issuer, validity, and fingerprints.

🧩

Components

Activities, services, receivers, and providers.

🔍

Strings

URLs, IPs, emails, API keys, and crypto addresses.

🎨

Light & Dark

Clean light theme by default. Toggle dark mode anytime.

Install

Get started

Clone the repository, run the installer, then use Scripts → Get APK Details on any APK file.

# Clone
git clone https://github.com/bhardwajAbhi/apk-inspector.git
cd apk-inspector

# Install
bash install.sh

# Manual test
python3 ~/.local/share/apk-inspector/apk_inspector.py /path/to/sample.apk

FAQ

Common questions

Quick answers for installation, usage, and platform support.

How do I access APK Inspector after installing?

Once install.sh has finished:

  1. Open Files (Nautilus)
  2. Navigate to any .apk file
  3. Right-click the file
  4. Hover over Scripts
  5. Click Get APK Details
Why don't I see "Get APK Details" under Scripts?

Re-run the installer and restart Nautilus:

bash install.sh
nautilus -q

Confirm the script exists:

ls -l ~/.local/share/nautilus/scripts/"Get APK Details"
Can I run it without the file manager?

Yes. Pass an APK path directly:

python3 ~/.local/share/apk-inspector/apk_inspector.py /path/to/sample.apk
Will Windows and Mac be supported?

Ubuntu / Nautilus is available today. Windows Explorer and macOS Finder integrations are on the roadmap.
